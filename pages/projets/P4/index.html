    Bienvenue sur la page dédiée au projet Ohmyfood. Ce projet est une réalisation dans le cadre de la formation OpenClassrooms. Vous avez intégré Ohmyfood en tant que développeur junior, une jeune startup qui souhaite s'imposer sur le marché de la restauration, d'abord à New-York et maintenant à Paris.

    Votre mission est de développer un site "mobile first" répertoriant les menus de restaurants gastronomiques. Les clients pourront composer leur menu et préparer leur repas pour éviter les temps d'attente au restaurant.

    L'équipe commerciale a déjà convaincu 4 restaurateurs d'utiliser la plateforme OhMyFood, et le site comprendra initialement 4 menus. Vous avez reçu des maquettes mobile et desktop ainsi qu'un prototype via Figma, incluant les animations et comportements à intégrer.

    Paul, le responsable du projet, a envoyé un e-mail de brief créatif avec des instructions détaillées. Le site doit être responsive, en "mobile first", et inclure des animations soignées. Il a hâte de voir le site terminé et est ouvert à une explication détaillée sur le fonctionnement des animations.

    Pour commencer, consultez les maquettes mobile et desktop ainsi que le prototype dans le dossier des sources du site (images et textes).

    Bon courage dans le développement du site Ohmyfood !

    Configuration de l'environnement de développement

    Avant de commencer le projet Ohmyfood, suivez ces étapes pour configurer votre environnement de développement.

    1. Clonez le répertoire source du projet à partir de GitHub : 
    git clone https://github.com/univers-mmc/ohmyfood.git
    1. Ouvrez le terminal intégré dans Visual Studio Code (VSCode). 
    npm init -y
    1. Ajoutez la commande suivante au package.json pour compiler Sass : 
    "scripts": {
                  "make-sass": "sass --watch build/sass.scss assets/styles.css"
                }
    1. Créez les répertoires et fichier nécessaires : 
    mkdir build assets &&&& cd build && touch sass.scss
    1. Déployez votre première version sur GitHub : 
    git add .
                git commit -m "Initial commit"
                git push -u origin main

    Découpage de la maquette

    Le découpage de la maquette est une étape cruciale dans le processus de développement d'un site web. Il s'agit de diviser la conception visuelle en éléments concrets, tels que les en-têtes, les menus, les sections de contenu, etc., prêts à être traduits en code HTML et CSS.

    Pour le site Ohmyfood, nous avons identifié les éléments principaux suivants :

    Le découpage de la maquette consiste à diviser la conception visuelle en éléments concrets, prêts à être traduits en code HTML et CSS. Voici les principaux éléments identifiés pour le site Ohmyfood :

    • En-tête (header) : Logo et barre de navigation.
    • Section Héros (main) : Bannière avec deux paragraphes captivants et un bouton animé.
    • Section Fonctionnement (main) : Trois boutons interactifs illustrant le fonctionnement.
    • Section Restaurants (main) : Quatre cartes de restaurants avec images, boutons "Nouveau", titres en gras, descriptions et icônes de cœur animées.
    • Pied de page (footer) : Icône Ohmyfood et logos de clients.

    Mise en place du Responsive Design

    Pour garantir une expérience utilisateur optimale, notre site web est conçu en utilisant la méthodologie "Mobile First". Cela signifie que nous commençons par concevoir pour les appareils mobiles, puis nous étendons les styles pour les tablettes et les ordinateurs de bureau.

    Voici quelques-unes des modifications apportées en fonction des breakpoints : 

    
                // ... Autres styles ...
                
                // Styles par défaut (mobile)
                .example-element {
                    // Styles spécifiques pour les appareils mobiles.
                }
            
                // Styles pour les tablettes (min-width: 600px)
                @media screen and (min-width: 600px) {
                    .example-element {
                        // Styles spécifiques pour les tablettes.
                    }
                }
            
                // Styles pour les ordinateurs de bureau (min-width: 1024px)
                @media screen and (min-width: 1024px) {
                    .example-element {
                        // Styles spécifiques pour les ordinateurs de bureau.
                    }
                }
            
                // ... Autres styles ...

    Animations en Sass

    Pour rendre le site Ohmyfood plus interactif et dynamique, nous avons intégré des animations en utilisant Sass, un préprocesseur CSS. Voici quelques-unes des animations que vous trouverez sur le site :

    • Loader Overlay : Un loader élégant avec un overlay qui s'affiche lorsqu'une action est en cours, offrant une expérience utilisateur fluide.
    • Cartes (Cards) : Des animations de transition et d'effet au survol des cartes de restaurants, apportant un aspect dynamique à la présentation.

      La palette du gout

       ménilmontant 
                                              
                                            /* Sass pour l'animation de la carte au survol */
                                            .card {
                                                /* Vos styles existants pour la carte */
                                                /* ... */
                                                
                                                /* Animation au survol */
                                                &:hover {
                                                    filter: brightness(1.2); /* Éclaircit la carte au survol */
                                                    .btn-new {
                                                        transform: translateX(10px); /* Déplace le bouton vers la droite */
                                                    }
                                                }
                                            }
    • Boutons : Des animations au survol (hover) des boutons pour ajouter une dimension visuelle agréable à l'interaction utilisateur.

    Composant Button

    Le composant button est utilisé pour créer des boutons interactifs sur le site. Il inclut des styles CSS pour un survol agréable et une utilisation cohérente sur toutes les pages. 

    
            .button {
                background-color: #9356DC;
                color: #fff;
                padding: 10px 20px;
                border: none;
                border-radius: 5px;
                cursor: pointer;
                transition: background-color 0.3s ease;
            
                &:hover {
                    background-color: lighten(#9356DC, 10%);
                }
            }

    Composant Loader

    Un élément essentiel pour améliorer l'expérience utilisateur est le composant "loader". Ce composant est une animation affichée au chargement de la page pour indiquer à l'utilisateur que le contenu est en cours de chargement.
